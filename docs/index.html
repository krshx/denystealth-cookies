Privacy Policy  ·  Last updated: February 2026

TL;DR — Guardr does not collect, store, or transmit any personal data whatsoever. It runs entirely inside your browser.

1. What Guardr Does

Guardr is a Chrome browser extension that automatically denies non-essential cookie consents and closes consent banners on websites you visit. It operates entirely within your browser and does not communicate with any external server by default.

2. Data We Do NOT Collect

  • We do not collect your browsing history or the URLs of sites you visit
  • We do not collect personal identifiers (name, email, IP address, device ID)
  • We do not use cookies, pixels, or any tracking technology
  • We do not store any information about which websites you use the extension on
  • We do not sell, share, or trade any data with third parties — ever

3. Optional Anonymous Telemetry Opt-in only

Guardr includes an entirely optional telemetry feature that is off by default. You must explicitly enable it in the extension settings. If you choose to enable it, the extension sends a single anonymous ping when a denial is performed, containing only:

  • A randomly generated session token — not tied to you, your device, or your account in any way, and regenerated each session
  • The type of consent management platform detected (e.g. "OneTrust" or "Cookiebot") — never the domain or URL of the site
  • The count of consents denied (e.g. "12")
  • The extension version number (e.g. "1.0.0")

This data contains no personally identifiable information. It is used solely to improve the extension by understanding which consent platforms are most commonly encountered. You can disable telemetry at any time via the ⚙ settings icon in the extension popup. Telemetry is off by default and must be actively switched on.

4. Permissions Explained

Guardr requests the following Chrome permissions, and uses each only for the purpose stated:

  • activeTab — to read and interact with the currently active tab when you click the extension button
  • scripting — to inject the consent-denial script into the active page
  • storage — to save your preferences (auto-deny mode, telemetry opt-in, donation reminder dismissed) locally on your device only
  • tabs — to read the current page URL for display inside the extension popup only; this is never transmitted anywhere
  • <all_urls> — required because cookie banners appear on all websites. Without this permission the extension cannot function. No data from any site is collected or transmitted as a result of this permission.

5. Local Storage Only

Your preferences (auto-deny on/off, telemetry opt-in, whether you dismissed the donation prompt) are stored using Chrome's storage.local API. This data lives only on your device and is never sent anywhere.

6. Single Purpose Statement

Guardr has a single purpose: to deny all non-essential cookie consents on your behalf, keeping only cookies that are technically necessary for sites to function. All permissions requested exist solely to enable this function.

7. Children's Privacy

Guardr does not knowingly collect data from anyone, including children under 13. The extension has no user accounts, no sign-in, and no age-gated features.

8. Changes to This Policy

If this privacy policy changes in a material way, the "Last updated" date above will be updated and changes will be noted in the Chrome Web Store listing. Your continued use of the extension constitutes acceptance of any updated policy.
